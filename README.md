Anaconda puede descargarse e instalarse de forma libre, incluyendo el uso comercial y la redistribución. Los códigos incluidos en este tutorial deberían ser compatibles con Python 2.7 y Python 3.4-3.6. Suponiendo que tengas Anaconda instalado, los siguientes comandos crean un entorno nuevo llamado `sklearn-env` e instalan todas las dependencias:```bashconda update condaconda update anacondaconda create --prefix ~/sklearn-env scikit-learnsource activate sklearn-envconda install matplotlibconda install ipythonconda install pandasconda install Pillow```### Preparación de equipos en la UCO (ThinkStation)Los equipos de la UCO están provistos de todo el material necesario para crear un entorno virtual donde trabajaremos durante todas las sesiones de los talleres.Un virtualenv o entorno virtual de Python es un ambiente creado con el objetivo de aislar recursos como librerías y entorno de ejecución, del sistema principal o de otros entornos virtuales. Lo anterior significa que en el mismo sistema, maquina o computadora, es posible tener instaladas múltiples versiones de una misma librería sin crear ningún tipo de conflicto.En primer lugar debemos crear un entorno virtual en nuestro home para nuestro taller, esto solo hay que hacerla una única vez, abrimos una terminal y tecleamos:```bash    cd    virtualenv taller_cdaa```De este modo queda creado nuestro entorno virtual en nuestro home con la siguiente estructura:```bash    taller_cdaa/        bin/        include/        lib/```A partir de ahora, cada vez que queramos trabajar con algo relacionado con nuestros talleres deberemos activar el entorno virtual con el comando:```bash    cd    cd  taller_cdaa/    source bin/activate    ```Ahora podrás ver en el prompt en la terminal como aparece el entorno virtual activado:```bash    (taller_cdaa)$  ```Para desactivar el entorno virtual solo deberás ejecutar el siguiente comando y verás que el prompt vuelve a la normalidad:```bash    (taller_cdaa)$  deactivate    $```Con el entorno virtual activado, nos vemos por la terminal hasta llegar al directorio raíz donde has descargado el repositorio de GitHub, y desde allí podemos instalar los paquetes necesarios para realizar el taller, ejecuta el comando:```bash    (taller_cdaa)$  pip installa requirements.txt```Con este comando se instalarán dentro del entorno virtual todas las bibliotecas necesarias para realizar correctamente todas las sesiones del taller.Recuerdo activar el entorno virtual cada vez que quieras trabajar en el taller.## Revisión de una correcta instalaciónTras obtener el material, **recomendamos encarecidamente** abrir y ejecutar el libro de notas ``check_env.ipynb``, que se encuentra en la raíz del repositorio.
